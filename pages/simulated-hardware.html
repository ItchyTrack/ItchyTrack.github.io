Simulated Hardware

This page showcases some simulated hardware components I built in the Scrap Mechanic game engine, which provides a set of six basic logic gates (and, or, xor, nand, nor, xnor) and a delay gate, plus expandability through math blocks for higher-level functions.

GPU

This is a simulated GPU with 64 threads that executes a pixel shader, utilizing math blocks for floating-point operations, coded in a minimal machine language. Each thread has its own block of memory and can read uniform variables.

These images show rendering two spheres with ray marching, rendering the Mandelbrot set, and the shader code to draw a circle.

CPU + Compiler

This is a small, register-less, simulated CPU supporting a variety of high-level math operations, including floating-point division, exponentiation, and square roots. The core of this project is a compiler, operating on a stack-free, C-like language to produce an intermediate representation, on which simple optimizations such as dead code elimination are performed, followed by memory layout and machine-code generation.

Floating Point ALU

Designed a gate-level, 16-bit floating point ALU with addition, subtraction, and comparison operations. The micro-architecture was optimized to reduce gate-count and latency while balancing pipeline depth to increase throughput.

Digit Recognizer Neural Network

Worked with two others to design and implement a three-layer perceptron entirely from low-level logic gates. Input from a 16x16 bi-level sensor passes through a hidden layer using ReLU activation, and feeds ten output neurons each representing a digit. Training was performed in Python, with weights quantized to four-bit signed integers and manually encoded into the hardware implementation.
